Pipeline For ETL

23.09.04 - 23.09.80

Introduction

CI/CD 환경을 구축하기 위해 Database 연동 및 자동화 과정은 꼭 필요한 과정 중 하나일 것이다. 그러한 과정을 이해하기 쉽고 효율적으로 구성하기 위해서는 Data Pipeline에 대한 이해가 기반이 되어야 할 것이다.

해당 프로젝트는 천재교육 빅데이터 4기 과정 중 김성은 강사님 수업을 토대로 진행했다.

Command 환경에서 Python Script에 인자를 넣어줄 수 있는 click 라이브러리를 통해 초기값으로 custom_batch_month(배치 작업 년월)와 job_name(배치 작업 시 수행되는 작업의 명칭)을 입력받는다.

• main.py : 입력 받은 custom_batch_month를 ‘%Y%m’형식으로 변환하여 _get_batch_month 함수로 return하며 _check_valid_month 함수로 해당 형태가 맞는지, 아니라면 Exception error를 반환한다.

• Database
∘ __init__.py : 해당 폴더를 패키지로 인식하기 위한 .py 파일
∘ connector.py : PostgrSQL, MongoDB에 접근할 수 있는 psycopg2, pymongo를 활용해 DB와의 직접적인 연결을 담당하는 파일 

import psycopg2 as pgsql
import pymongo as mongo

class DBConnector:
def __init__(self, host, port, user, pw, db, location, engine):
    self._host = host
    self._port = port
    self._user = user
    self._pw = pw
    self._db = db
    self._location = location
    self._engine = engine
    self.conn = None

    if location == 'localhost_source':
        if engine == 'postgre':
            self.enter_connect = self.pgsql_connector
        elif engine == 'mongodb':
            self.enter_connect = self.mongo_connector
        else:
            raise RuntimeError(f"{engine} is not supported")

    elif location == 'localhost_target':
        if engine == 'postgre':
            self.enter_connect = self.pgsql_connector
        elif engine == 'mongodb':
            self.enter_connect = self.mongo_connector
        else:
            raise RuntimeError(f"{engine} is not supported")

    else :
        raise RuntimeError(f"{location} is not supported")

def __enter__(self):
    self.enter_connect()
    print('<< DBConnection >>')
    print('\tConnection Info : ', self.conn)
    return self

def __exit__(self, exc_type, exc_val, exc_tb):
    try :
        self.conn.close()
    except:
        pass
    # self.conn.close()
    # self.get_collection.close()

def pgsql_connector(self):
    print('<< pgsql_connector >>')
    self.conn = pgsql.connect(host = self._host, port = self._port, user = self._user, password = self._pw, dbname = self._db)

def mongo_connector(self):
    print('<< mongo_connector >>')
    self.conn = mongo.mongo_client.MongoClient(host = self._host, port = self._port, username = self._user, password = self._pw).get_database(self._db)

• pipeline
∘ controller.py : main.py에서 호출되는 모듈로써 extract.py, load.py, transform.py 모듈의 기능을 종합하여 컨트롤 하는 모듈
∘ extract.py : Database에서 데이터를 추출하는 모듈
∘ load.py : Database에 데이터를 적재하는 모듈
∘ transform.py : controller를 통해 입력받은 result 값을 병합하고 dataframe 형식으로 변환시켜주는 모듈. Document Database에서 데이터를 가져와 Relational Database에 적재할 때 사용하거나, Relational Database의 데이터를 변환할 때 사용


Discussion

복잡한 인물관계를 가진 소설책을 읽을 때 인물들 간의 관계가 헷갈리면 그들의 관계를 직접 손으로 그려보곤 했다. 이와 비슷하게 Pipeline 프로젝트 때도 시각화를 통해 각 모듈들의 필요성과 관계를 더 쉽게 파악할 수 있었다.

Python 스크립트를 한 파일에 넣지 않고 기능별로 구분하여 모듈로써 활용하는 것은 효율성 면에 있어서 꼭 필요한 작업일 것이다. 클래스 활용, 패키지화, 모듈화의 필요성에 대해 인지할 수 있 프로젝트였다.
