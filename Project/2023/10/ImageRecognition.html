Introduction

코딩 교구는 디지털 시대로의 변화에 대비하는데 중요한 역할을 하기 때문에 어린 학생들의 미래를 준비하고 다양한 학습 경험 제공을 위해 필요가 있다고 느꼈다.

그러한 코딩 교구 제작에 있어서 스마트폰을 활용한 어플리케이션이 접근성이 쉽다고 판단했고, 이미지 인식 기능이 있는 Mobile Application을 구현해 보는 것을 목표로 잡았다. 또한 실제 Mobile Application이 만들어지고 배포되는 전반적인 process에 대한 탐구에도 목적이 있다.

해당 프로젝트는 OSMO 코딩 교구를 reference로 한다.


Scheduling

기획 회의는 팀원들 모두 참여해 결정했다.
나를 포함한 팀원들 모두 Mobile Application 제작 경험이 전무했기에 초기에는 제작 방법에 대해 팀원들과 같이 고민했다. 후에 나머지 팀원들은 이미지 인식 모델 개발을 메인으로 맡아줬고 나는 Mobile Application에 어떤 DB를 사용할지, 어떻게 Server와 연동할 수 있을지 공부하며 작업했다.

초기에는 Kivy, Beeware 등 Python으로 App을 만드려고 시도했지만, 환경 설정 및 버전 호환 문제에 부딪혀 실패했고 많은 시간을 소모했다. 고민 끝에 새로운 언어를 시도해보기로 마음 먹었고, 최종적으로 Flutter를 사용했다.

코딩 교구를 제작하는 프로젝트인 만큼 콘텐츠를 추가하는 등의 고도화도 진행하려 했지만 한정된 프로젝트 기간과 예상보다 길었던 기술 탐구로 Prototype까지만 구현됐다.


Database

빠른 Prototype 도출을 위해 Flutter 프레임워크를 선택했고, Database 구축 및 연동에는 웹앱 플랫폼에 대해 클라우드 호스팅 서비스를 제공하는 Firebase를 활용했다.

1. Cloud Firestore

빠른 쿼리와 실시간 동기화 기능을 제공하는 NoSQL Database. Realtime Database에 비해 다양한 기능이 있다고 하지만, 제작한 모바일 앱을 Virtual Machine이 아닌 개인 기기에서 실행했을 때 데이터 적재가 느린 경우가 있어 실험 단계에서만 사용했다.

2. Realtime Database

Firebase 초기 Database이지만, 모바일 앱 내에서 찍은 사진이 실시간으로 서버를 거쳐 나와야 하는 상황에서는 실험 결과 속도가 더 빨랐기 때문에 최종적으로 Realtime Database를 선택했다.


Server

AWS EC2 서비스를 이용하여 Ubuntu 서버를 만들었고, 이미지 인식 모델과 Firebase의 Database에 접근하는 코드가 포함된 Python 파일을 FastAPI로 Serving 했다.


Result

최종적으로 구현된 Process는 아래와 같다. 1. App → DB
사용자가 App 내에서 사진을 찍으면 해당 사진은 Firebase DB에 적재된다.

2. DB → Server (Modeling)
DB에 사진이 적재되면, AWS EC2 server에 있는 Image Modeling File은 해당 사진으로 결과값을 도출한다.

3. Server → App
Server로부터 결과를 전송받으면 App 내의 캐릭터는 인식한 결과대로 이동한다.

User Guide

1. 교구를 준비한 뒤 어플을 실행한다.
2. 좌측 하단 ‘카메라’ 모양의 버튼을 누른다
3. 준비한 교구 사진을 찍는다
4. Loading : 이미지가 Database에 적재되는 시간 (최대 약 5초)
5. 로딩이 완료되면 ‘★JUUNS★’ 또는 ‘♥YOLO♥’ 버튼을 클릭한다
→ 이미지가 서버 모델을 거쳐 결과가 나오는 시간, 최대 약 3초
6. 인식한 이미지와 같은 움직임인지 확인한다

📌 본 어플은 Osmo Coding의 교구 데이터로 제작됨
📌 ‘루돌프’는 상, 하, 좌, 우 그리고 2~5의 숫자를 인식함 (Motion Block 인식 불가)
📌 Error Message가 뜨면 화살표를 인식하지 못한 것
📌 모델을 눌렀는데 아무런 반응이 없다면 사진을 다시 찍고 시도하기!


Result

Database 적재와 Server Modeling을 거쳐 캐릭터가 이동하는 것을 확인할 수 있다.



개선 방향
• 교육 콘텐츠 기획
현재는 이미지 인식 후 캐릭터 이동만 가능한 상황인데, 참고한 Osmo 코딩 교구처럼 다양한 콘텐츠를 도입하여 개발할 수 있다.
• 지연 시간 최소화
Image Modeling 결과값이 도출되기 까지 약 4-5초 정도의 로딩 시간이 걸리는데, 빠른 상호작용을 위해 2초 이내로 작동할 수 있도록 기능을 최적화할 수 있다.
